# JVO Agent Manager — Прототип

Кликабельный lo-fi прототип продукта «агент-планировщик» для демо стейкхолдерам.
Продукт — B2B AI-платформа для продавцов на Wildberries/Ozon (JVO / Дживио).

## Запуск

```bash
npm install      # один раз
npm run dev      # локальный сервер на http://localhost:5173
npm run build    # сборка в ./dist
npm run preview  # превью собранной версии
```

## Архитектура jvo-prototype.jsx

### Экраны (screens)

Навигация реализована через `useState` в компоненте `App` — никакого роутера нет.

| screen        | Компонент         | Описание                                               |
|---------------|-------------------|--------------------------------------------------------|
| `start`       | `StartScreen`     | Стартовая страница с инпутом задачи                    |
| `chat`        | `ChatScreen`      | Диалог с агентом + панель анализа                      |
| `plans`       | `PlansScreen`     | Список планов (одна карточка)                          |
| `dashboard`   | `DashboardScreen` | Дашборд плана (табы: план, сценарии, события)          |
| `report`      | `ReportScreen`    | Итоговый отчёт по плану                                |

Переходы между экранами: `setScreen(...)` из `App`.

### Компоненты внутри файла (сверху вниз)

```
GLOBAL_STYLES          CSS-анимации (@keyframes breathe, fadeInUp)
A                      константа с цветами акцента (purple)

REASONING_LINES        данные: строки рассуждений агента в панели анализа
INSIGHT_CARDS          данные: инсайт-карточки в панели анализа
SCENARIO_CANDIDATES    данные: кандидаты сценариев (2 отклонённых, 3 принятых)
CHAT_SCRIPT            данные: полный скрипт диалога (18 шагов)
SCENARIOS_PROPOSAL     данные: сценарии для экрана «Запустить план»
SCENARIOS_DASHBOARD    данные: сценарии для дашборда (с corrections)
EVENTS                 данные: события для таба «События»
CHART_DATA             данные: план/факт/база для графиков
REPORT_ROWS            данные: строки таблицы в отчёте

Sidebar                боковая навигация (иконки)
StartScreen            стартовый экран
ChatScreen             чат + управление авто-воспроизведением
  UserBubble           пузырь пользователя
  AgentMessage         сообщение агента (включая summary-карточку)
  TypingDots           три точки «печатает»
AnalysisPanel          панель анализа матрицы (Фича 1)
WatchingBadge          пульсирующий бейдж «Агент ведёт план» (Фича 2)
ScenarioToast          тост «Сценарий скорректирован» с авто-показом (Фича 3)
PlansScreen            список планов
DashboardScreen        дашборд плана (хост для трёх табов)
  PlanTab              таб «План» — метрики, график
  ScenariosTab         таб «Сценарии» — список + детали + корректировки
  EventsTab            таб «События» — таблица событий + фильтр
ReportScreen           итоговый отчёт
DemoPanel              плашка внизу для быстрого перехода между экранами
App                    корневой компонент, управляет экраном
```

### Ключевые механики

**Авто-воспроизведение чата** — `ChatScreen` через `useEffect` + `setTimeout` пошагово показывает сообщения из `CHAT_SCRIPT`. Агентские шаги — 1300ms задержка, пользовательские — 700ms. Кнопка «Пропустить диалог» показывает всё сразу.

**Панель анализа (AnalysisPanel)** — собственный таймер (`step` 0→17→99). Строки рассуждений появляются с каскадной прозрачностью (новая = 1.0, старше — 0.6, 0.4, 0.25). Отклонённые сценарии схлопываются через CSS `max-height` анимацию. Кнопка «Пропустить» переходит сразу в step=99.

**Мутация сценариев** — `DashboardScreen` держит `scenarios` в `useState` (инициализируется из `SCENARIOS_DASHBOARD`). «Подтвердить» корректировку вызывает `onConfirmCorrection(scenarioIdx, correctionIdx)` — убирает корректировку из массива и показывает тост «Корректировка подтверждена».

**Тоггл «Публиковать без подтверждения»** — при выключении открывает модальное окно с предупреждением. Модаль поддерживает `onConfirm`, `confirmLabel`, `cancelLabel`.

**Тост Фича 3** — показывается один раз через 14 секунд после открытия дашборда (через `toastShown = useRef(false)`), автоматически скрывается через 6 секунд.

**DemoPanel** — фиксированная плашка внизу. Функция `jumpTo(target)` в `App` обрабатывает все переходы, включая сброс ключей (`chatKey`, `dashKey`) для форс-ремаунта компонентов.

### Данные: SCENARIOS_DASHBOARD

Главный массив с 5 сценариями. Каждый сценарий:
```js
{
  id, name, icon, agent, desc, algorithm, impact, coverage, freq,
  corrections: [
    {
      date, time, text,
      changes: [{ label, from, to }]
    }
  ]
}
```
Корректировки сценариев 1 и 2 заполнены, остальные — пустые массивы.

Акцент — `purple-600` (`#7c3aed`). Объект `A` содержит все нужные классы:
```js
const A = {
  bg: 'bg-purple-600',

  light: 'bg-purple-50',
  lightText: 'text-purple-700',
  ring: 'focus:ring-purple-500',

```

## Что работает в прототипе

- Полный флоу: Старт → Чат → Анализ → Дашборд → Отчёт
- Авто-воспроизведение диалога с анимацией
- Панель анализа матрицы с поэтапным появлением контента
- Подтверждение корректировок сценариев с тостом
- Тоггл «Публиковать без подтверждения» с модальным предупреждением
- Info-кнопка с попапом
- Inline-редактирование названия плана (карандаш Edit3)
- Фильтр событий по статусу (шестерёнка в таблице событий)
- «Опубликовать все» / «Пропустить все» в обоих табах — модальное подтверждение
- Быстрая навигация через DemoPanel (плашка внизу экрана)
